# Чек-лист для понимания проекта

Перед сдачей попробуйте ответить своими словами:

1. За что отвечает файл `models.py`?
2. Почему заявки связаны с объектом через `ForeignKey`?
3. Где в проекте выполняется фильтрация по цене?
4. Как Django защищает форму от CSRF?
5. Почему файл `db.sqlite3` не нужно добавлять в GitHub?
6. Чем ветка `feature/...` отличается от `main`?

Мини-практика: добавьте в модель `Property` поле `rooms` — количество комнат, выполните миграции и выведите поле в шаблоне карточки.
